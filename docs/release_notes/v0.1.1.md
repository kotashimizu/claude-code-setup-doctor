# リリースノート — v0.1.1-beta

**リリース日**: 未定（テスト完了後）

---

## このリリースについて

v0.1.0 の実機検証（志水）で判明した以下の問題を修正し、未実装だった主要機能を追加しました。

## 修正した不具合

- 「診断結果を保存」を押しても実際にはファイルが出力されず、押した後に画面から戻れなくなる不具合を修正。実際にJSON/テキストレポートを保存し、保存先を開くボタンを追加。
- 診断結果画面に「自動で直す」ボタンが存在せず、修復機能（バックエンドは実装済み）がUIから一切利用できなかった問題を修正。修復内容の確認画面・実行画面を新規実装。
- タイトルバーで「（非公式）」表記が2箇所に重複表示されていた問題を解消。

## 追加した機能

- **修復レビュー画面**: 自動修復の対象・変更前後・再起動要否を確認し、選択して実行できる。
- **修復実行画面**: 各修復アクションの進捗を表示し、完了後は自動で再診断する。
- **完了画面**: 修復後の状態に応じて「利用可能」「新しいターミナルが必要」等を案内し、再診断・レポート保存・スタートへ戻るを提供。
- **非技術者向けの結果表示**: チェックIDや内部コードをそのまま出す代わりに、平易な日本語の一言説明と「◯項目中◯項目は問題ありません」という要約を画面上部に表示。技術的な詳細は折りたたみで別途閲覧可能。

## 既知の制限

- Windows PowerShell がPATHに見当たらない場合（CHK-SHELL-001のRepairable状態）の自動修復は未実装（`docs/implementation_notes/open_questions.md` Q-008）。
- REM-AUTH-LOGIN（ブラウザログイン起動）、REM-INSTALL-CLAUDE-WINGET は引き続き未実装。
- 専用のAbout画面は未実装（非公式である旨の開示はスタート画面の説明文のみ）。
