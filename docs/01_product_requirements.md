# 01. Product Requirements Document

## 1.1 ビジョン

Claude Codeの導入時に発生するWindows固有の環境差を、利用者が理解できる診断結果と、影響範囲を限定した修復手段に変換する。

## 1.2 プロダクト目標

### Goal G-1: 自己診断

利用者が支援者の同席なしで、Claude Codeが使えない主因を特定できる。

### Goal G-2: 安全な自己修復

既知のユーザー領域の問題は、変更内容を確認した上で修復できる。

### Goal G-3: 適切なエスカレーション

企業ポリシー、ネットワーク、証明書、EDRなど、アプリが修復してはいけない問題を明確に区別し、診断情報を情シスへ渡せる。

### Goal G-4: 導入手順の標準化

支援者が顧客ごとに異なる口頭説明をせず、同一の画面・判定・レポートで対応できる。

## 1.3 成功指標（プロダクト目標値）

以下は実績値ではなく、パイロットで検証する目標です。

- 必須チェックの結果が、1回の診断で「利用可能／修復可能／IT対応」のいずれかに分類される。
- 自動修復後に再診断し、修復の成否を利用者へ明示する。
- 診断レポートに秘密情報が含まれないことを自動テストで保証する。
- 代表的な環境差シナリオを自動テストまたはVM手動テストで再現できる。
- 必須操作がキーボードだけで完了できる。

## 1.4 ユーザーストーリー

### US-001 初回診断

利用者として、ボタンを1つ押してClaude Codeの利用可否を確認したい。そうすれば、ターミナル操作を理解していなくても次の行動が分かる。

### US-002 PATH修復

利用者として、Claude Codeがインストールされているのにコマンドが見つからない場合、追加されるPATHを確認して修復したい。

### US-003 Git Bash設定

Git利用者として、Gitがカスタムフォルダに存在する場合、Claude Code専用設定へGit Bashパスを登録したい。

### US-004 認証

利用者として、未ログインであることを確認し、公式のログインフローを開始したい。

### US-005 IT引き継ぎ

企業利用者として、自動修復できないネットワークやセキュリティ制限を、情シスに渡せるレポートとして保存したい。

### US-006 導入支援

導入支援者として、利用者の画面共有なしで診断レポートを受け取り、原因を判断したい。

## 1.5 MVPスコープ

### Must

- サポート対象OS／64-bit／メモリの判定
- PowerShellまたはGit Bashの検出
- Claude CLIの検出、実体パス、バージョン確認
- `%USERPROFILE%\.local\bin` の存在とUser PATH判定
- 複数 `claude` およびWindowsApps競合の検出
- `claude auth status` による認証状態判定
- GitおよびGit Bashの検出（推奨項目）
- User PATHへのClaudeネイティブパス追加
- `CLAUDE_CODE_GIT_BASH_PATH` のユーザー設定追加
- ログインコマンドの可視ターミナル起動
- 修復前確認、バックアップ、再診断
- JSON／テキストレポート
- 日本語UI

### Should

- WinGet検出およびClaude Codeインストール起動
- ネットワーク到達性テスト
- `claude doctor` の補助診断
- Desktop App Execution Alias競合案内
- 診断結果のコピー
- 詳細ログの展開表示

### Could

- ARM64配布
- MSIX／MSI配布
- 管理者向けサイレント診断モード
- 企業管理設定の読み取り表示
- 英語UI
- オンライン更新通知

### Won't in MVP

- 管理者権限での自動修復
- Machine PATH変更
- レジストリポリシー変更
- GPO、AppLocker、EDR、証明書、プロキシ設定変更
- Claude Desktopの自動アンインストール／更新
- npm版のインストール支援
- 任意プラグインやMCPの診断

## 1.6 全体状態

全体状態は次のいずれかです。

| 状態 | 条件 | 利用者への表示 |
|---|---|---|
| Ready | 必須チェックがすべてPass | Claude Codeを利用できます |
| ReadyWithRecommendations | 必須はPass、推奨項目にWarning | 利用できます。一部の推奨設定があります |
| Repairable | 必須項目に、アプリで安全に修復できるFail | 修復すると利用できる可能性があります |
| UserActionRequired | 認証や再起動など利用者操作が必要 | 追加操作が必要です |
| ITActionRequired | 管理領域、ネットワーク、EDR等が疑われる | 情報システム担当者へ確認してください |
| Unsupported | OS／アーキテクチャ／メモリが要件外 | この環境はサポート対象外です |
| Unknown | タイムアウト等で判定不能 | 状態を確認できませんでした |

## 1.7 制約

- Claude Codeの出力文言やパスは更新され得るため、テキスト解析への依存を最小化する。
- 企業端末ではWinGet、PowerShell、子プロセス、設定ファイル編集がポリシーで制限される場合がある。
- DesktopアプリとCLIは同じ基盤でも、インストール形態とPATHは同一とは限らない。
- 認証済みでも契約・組織権限・サービス障害により実行できない場合がある。
- 本ツールはAnthropicの公式サポートを代替しない。
