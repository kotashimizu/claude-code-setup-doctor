# START HERE - Claude Codeへの引き継ぎ

## 1. プロダクト概要

**仮称:** Claude Code Setup Doctor  
**表示名:** Claude Code セットアップ診断ツール（非公式）

Windows利用者が、Claude Codeをローカルで利用できる状態かをワンクリックで確認し、原因と安全な修復手段を理解できるアプリを作ります。

このアプリはClaude Codeの代替UIではありません。コード編集、チャット、ターミナル統合は行わず、セットアップ診断と限定的な修復に特化します。

## 2. MVPの到達点

利用者がアプリを起動し、次の流れを完了できることをMVPの完了条件とします。

1. 「診断を開始」を押す。
2. OS、シェル、Claude CLI、PATH、認証、Gitの状態が表示される。
3. 問題がある場合、原因、影響、修復内容が日本語で表示される。
4. 利用者が修復内容を確認して承認する。
5. アプリが許可済みの修復だけ実行し、再診断する。
6. 「利用可能」「一部機能のみ利用可能」「IT管理者の対応が必要」のいずれかが明示される。
7. 個人情報と秘密情報を除外した診断レポートを保存できる。

## 3. 技術方針

- .NET 10 LTS
- WPF
- C#
- MVVM
- 自己完結型・単一ファイル配布を基本とする
- x64をMVP、ARM64は後続
- 管理者権限なしで診断
- 修復はユーザー領域に限定
- OS操作はインターフェース化し、単体テストでは実環境を変更しない

## 4. 実装開始時の必須ルール

- 最初に `CLAUDE.md` と `docs/` を読み、実装計画を作成する。
- 実装前に、仕様間の不一致を一覧化する。
- 診断と修復を同じクラスに混在させない。
- 実環境のPATH、レジストリ、Claude設定を単体テストから変更しない。
- 外部コマンドは固定された実行ファイルと引数配列だけを使用する。シェル文字列連結は禁止。
- ログにトークン、APIキー、環境変数値、完全なユーザー名を出力しない。
- 修復前にバックアップと差分プレビューを作成する。
- エラー時に「成功」と表示しない。再診断で確認できた場合だけ成功とする。

## 5. 最初の作業

`prompts/01_bootstrap_repository.md` をClaude Codeに渡し、Phase 0だけを実行します。その後、ビルドとテストが通ったことを確認してからPhase 1へ進みます。
